# GroundClock Temporal Context — Spec v0

A small, explicit contract for the "now" an application supplies to an LLM. The goal is a
*consistent, timezone-aware, provenance-tagged* temporal context that (a) renders to a stable
system-prompt block and (b) is also available to the model as a `get_current_time` tool, so a
benchmark can measure whether the model actually uses it.

## Temporal Context object

A JSON object. Machine-readable schema: `../../src/groundclock/schemas/temporal_context.schema.json`.

| Field | Type | Required | Description |
|---|---|---|---|
| `instant` | string (RFC 3339, UTC, `Z`/`+00:00`) | yes | The reference instant in UTC. The single source of truth. |
| `timezone` | string (IANA tz name) | yes | The user/session timezone, e.g. `America/New_York`. |
| `local_time` | string (RFC 3339 with offset) | yes | `instant` rendered in `timezone`. Must equal `instant` when converted. |
| `utc_offset` | string (`±HH:MM`) | yes | Offset of `local_time` from UTC at `instant` (DST-correct). |
| `weekday` | string | yes | Day name of `local_time` (e.g. `Friday`). |
| `sequence` | integer ≥ 0 | yes | Monotonic counter within a session. Lets a benchmark detect drift/inconsistency across turns. |
| `provenance` | enum | yes | Origin of `instant`: `system_clock`, `injected`, `tool`, `unknown`. |
| `knowledge_cutoff` | string or null | no | The model's training cutoff (e.g. `2026-01`), held *separately* from `instant`. |

### Invariants

1. `local_time` converted to UTC MUST equal `instant` (no skew).
2. `utc_offset` MUST be the actual offset of `timezone` at `instant` (DST-aware).
3. `sequence` MUST be non-decreasing within a session; a benchmark MAY treat a decrease or a
   backwards `instant` as an inconsistency failure.
4. `provenance` MUST reflect the true origin. A model answering "now" from `knowledge_cutoff`
   instead of `instant` is the failure mode NowBench measures.
5. `knowledge_cutoff` is advisory context, never the answer to "what is the current date".

## Rendered system-prompt block

The object renders to a deterministic text block injected into the system prompt:

```
<temporal_context>
Current date and time: 2026-06-20T14:30:00-04:00 (Friday)
Timezone: America/New_York (UTC-04:00)
Reference instant (UTC): 2026-06-20T18:30:00+00:00
Provenance: system_clock | sequence: 3
Knowledge cutoff: 2026-01. For anything after this date, use the current date above or call get_current_time.
</temporal_context>
```

Rendering MUST be byte-stable for a given object (so it is prompt-cache friendly — the volatile
instant is the only thing that changes; see Anthropic prompt-caching guidance).

## `get_current_time` tool

Schema: `../../src/groundclock/schemas/get_current_time.tool.json`. Input: optional
`timezone` (IANA name) to convert into; returns the Temporal Context object as JSON. The tool
and the injected block draw from the **same** clock, so they never disagree.

## Versioning

`v0` is unstable. Breaking changes bump to `v1`. New optional fields are non-breaking.
